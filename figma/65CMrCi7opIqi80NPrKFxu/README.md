## Extracted Theme Tokens

Canvas:
- 16:9, `1920 x 1080`.
- Primary content margin is about `100px` left/right for chrome and most content.
- Header chrome commonly uses project name at `x=100 y=46`, page number near `x=1793 y=46`, and a thin horizontal rule at `y=100`.

Colors:
- Lime accent/background: `#c7ef4e`.
- Dark green background: `#003310`.
- Metric green: `#085420`.
- Black: `#000000`.
- White: `#ffffff`.
- Secondary text gray: `#7a7a7a`.
- Light neutral placeholders/checker areas appear on image-heavy slides.

Typography:
- Display/title serif: `Neuton Light`.
- Body/UI sans: `Open Sans Regular` and `Open Sans SemiBold`.
- Large display: `Neuton Light`, `200px`, `line-height 100%`, tight negative tracking around `-2.2%`.
- Standard slide title: `Neuton Light`, `80px`, `line-height 110%`, tracking around `-2%`.
- Body large: `Open Sans Regular`, `36px`, `line-height 120%`, tracking around `-2.2%`.
- Body medium: `Open Sans Regular`, `24px`, `line-height 130%`.
- Chrome/labels: `Open Sans SemiBold`, `20px`, uppercase, `line-height 120%`, tracking around `2%`.

Shape and layout language:
- Mostly flat, editorial, grid-based composition.
- No rounded card system in the sampled frames.
- Frequent use of thin 1px dividers, horizontal rules, vertical separators, and long timeline rules.
- Lime blocks are used as both full-bleed backgrounds and hard-edged data/card fields.
- Dark pages are sparse, high-contrast, and use lime serif display type for emphasis.
- White pages use restrained lime pills, progress bars, and thin lines instead of heavy cards.

Cookbook-relevant layout families:
- Full-bleed lime divider/agenda.
- White metric grid with lime data panels.
- Lime multi-column action plan.
- Lime image-card milestone page.
- Black or dark-green title/overview pages.
- White status dashboard with pill labels and progress bar.
- White horizontal timeline.
- White budget page with right-side serif metric stack.
- Black risk/mitigation rows with lime connectors.

Notes for future cookbook revisions:
- Treat these frames as visual grammar, not as a closed layout whitelist.
- Preserve full chart-catalog freedom; cookbook should restyle charts into this visual system.
- Preserve density modes because the samples include sparse title pages, medium dashboard pages, and dense risk/action text pages.
